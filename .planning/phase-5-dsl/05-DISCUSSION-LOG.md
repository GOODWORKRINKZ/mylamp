## Модель анимации

| Option | Description | Selected |
|--------|-------------|----------|
| Расширить sprite кадрами | Добавить в DSL синтаксис frames внутри sprite: каждый кадр — свой bitmap. Layer получает поле frame с выражением | ✓ |
| C++ скомпилированный эффект | Написать Mario как IEffect на C++. Полный контроль, анимация вшита. Не расширяет DSL | |
| Несколько спрайтов + use-выражение | Объявить mario_0, mario_1, mario_2 как отдельные спрайты. Разрешить выражение в layer.use | |

**User's choice:** Расширить sprite кадрами
**Notes:** Пользователь хочет, чтобы анимация была частью DSL, а не жёстко вшита в C++. Это сохраняет live-coding философию.

## Расширения DSL

| Option | Description | Selected |
|--------|-------------|----------|
| Кадры + выражение в use | Именованные кадры в sprite и выражение для выбора кадра в layer | ✓ |
| Ещё и циклы for | Добавить for для fullscreen-эффектов без bitmap 32×16 | ✓ |
| Ничего, кроме кадров | Текущих выражений sin/cos/clamp/smoothstep достаточно | |

**User's choice:** Кадры + use-выражение + циклы for
**Notes:** Пользователь сначала хотел проверить, можно ли Nyan Cat сделать без циклов. Анализ показал, что шлейф возможен через fullscreen-слой с per-pixel выражениями (smoothstep), но циклы for сильно упрощают fullscreen-эффекты и делают DSL выразительнее. Решили добавить.
