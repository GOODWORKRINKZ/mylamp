## UI Element Inventory

### 1. Share Button — NEW

**Location:** Editor toolbar, after "Сохранить" button.
**Current toolbar order:** `Новый | Проверить | Запустить | Сохранить`
**New order:** `Новый | Проверить | Запустить | Сохранить | Поделиться`

**Markup (insert into `shellTemplate.ts` after `save-button`):**
```html
<button id="share-button" type="button">Поделиться</button>
```

**Styling:** Inherits all `<button>` styles from `app.css`:
- `border-radius: 999px` (pill)
- `background: var(--accent)` (#d6552f)
- `color: #fff7f1`
- `padding: 0.5rem 0.85rem`
- `font-size: 0.85rem`
- Hover: `translateY(-1px)` + `box-shadow: 0 8px 18px rgba(214,85,47,0.2)`
- Gap from adjacent buttons: `0.4rem` (inherited from `.editor-bar__actions`)

**No new CSS rules needed.**

**Behavior (implemented in `main.ts`):**
1. On click: get editor content via `getEditorValue()`
2. Encode as URL-safe base64 (RFC 4648 §5):
   - Standard `btoa()` → replace `+` with `-`, `/` with `_`, strip `=` padding
3. Build share URL: `${window.location.origin}${window.location.pathname}?code=${encoded}`
4. Copy to clipboard: `navigator.clipboard.writeText(shareUrl)`
5. On success: set `#editor-status` text to "Ссылка скопирована. Отправь её кому угодно."
6. On error: set `#editor-status` text to "Не удалось скопировать ссылку."

### 2. Share Link Decoding on Load — NEW BEHAVIOR

**No new UI.** On page load, if `window.location.search` contains `?code=...`:
1. Decode URL-safe base64 → standard base64 → `atob()` → editor content
2. Load decoded source into editor via `setEditorValue()`
3. Set status: "Эффект загружен по ссылке. Можно запустить или сохранить."

**URL format:**
```
http://<lamp-ip>/?code=<url-safe-base64>
```

### 3. Save Button Route Fix — EXISTING, VISUAL UNCHANGED

**No visual change.** The `#save-button` already exists in the toolbar. The fix is purely in `main.ts` `savePreset()` function:

**Before (broken):**
```ts
const response = await fetch(`/api/presets?id=${encodeURIComponent(presetId)}`, {
  method: "PUT",
```

**After (fixed):**
```ts
const response = await fetch(`/api/presets/${encodeURIComponent(presetId)}`, {
  method: "PUT",
```

Success/error status messages remain unchanged:
- Success: `editor-status` = "Сохранение завершено."
- Error: `editor-status` = "Ошибка при сохранении."
