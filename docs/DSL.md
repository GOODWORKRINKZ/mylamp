# Lux Language Reference

Lux — это язык эффектов для `mylamp` (от лат. *lux* — свет). Он специально сделан маленьким: язык должен быть понятным человеку, предсказуемым для LLM и достаточно жёстким для последующей реализации на ESP32-C3.

## Зачем он нужен

Lux нужен, чтобы:

- быстро собирать визуальные эффекты без правки firmware на C++;
- держать контракт между frontend, mock backend и device-side runtime;
- ограничивать синтаксис, чтобы генерация через LLM не расползалась в выдуманный язык.

## Базовые сущности

Lux опирается на четыре конструкции:

- `effect`
- `sprite`
- `text`
- `layer`

## `effect`

Каждый файл эффекта начинается с имени:

```txt
effect "rainbow"
```

Это имя:

- показывается в UI;
- участвует в сохранении preset-а;
- служит главным идентификатором эффекта.

## `sprite`

`sprite` описывает форму через bitmap-маску.

Пример:

```txt
sprite heart {
	bitmap """
	.##.##.
	#######
	#######
	.#####.
	..###..
	...#...
	"""
}
```

Смысл символов:

- `#` — закрашенный пиксель
- `.` — пустой пиксель

## `text`

`text` генерирует спрайт из строки символов с помощью встроенного 3×5 пиксельного шрифта.

Синтаксис:

```txt
text <имя> "<содержимое>"
```

Пример:

```txt
text greeting "HELLO"
```

После объявления `text` можно использовать в `layer` через `use`, точно так же как обычный `sprite`:

```txt
layer label {
	use greeting
	color rgb(255, 255, 255)
	x = 2
	y = 5
	scale = 1
	visible = 1
}
```

Поддерживаемые символы: A–Z, А–Я (регистронезависимо), 0–9, пробел, `!`, `.`, `,`, `:`, `-`, `+`, `?`.
Кириллические буквы, совпадающие по форме с латинскими (А=A, В=B, Е=E, К=K и т.д.), используют общие глифы. Все 33 буквы русского алфавита поддерживаются.

Каждый символ — 3 пикселя в ширину с 1-пиксельным промежутком. Высота — 5 пикселей.

## `layer`

`layer` выбирает sprite и задаёт его поведение.

Пример:

```txt
layer love {
	use heart
	color rgb(255, 40, 80)
	x = 10 + sin(t) * 4
	y = 4 + cos(t * 1.2) * 2
	scale = 1
	rotation = t * 0.5
	blend = add
	visible = 1
}
```

Поддерживаемые поля слоя:

- `use` — имя `sprite` или `text`
- `x`
- `y`
- `scale`
- `rotation`
- `blend`
- `color`
- `visible`

`rotation` задаётся в радианах и вращает sprite вокруг центра его bitmap.
`blend` задаёт режим наложения слоя: `normal`, `add`, `multiply`.

## Доступные переменные

- `t` — время в секундах с момента старта эффекта
- `dt` — дельта времени между кадрами
- `x`, `y` — текущие координаты пикселя
- `nx`, `ny` — нормализованные координаты от `0` до `1`

## Доступные функции

Математика:

- `sin(value)`
- `cos(value)`
- `abs(value)`
- `min(a, b)`
- `max(a, b)`
- `clamp(v, a, b)`
- `mix(a, b, t)` — линейная интерполяция
- `smoothstep(edge0, edge1, v)` — гладкий порог

Случайные:

- `random(max)` — случайное число в `[0, max)`. Разное для каждого пикселя.
- `randf(max)` — случайное число в `[0, max)`. Одно на весь кадр (f = frame).

Условия:

- `if(cond, a, b)` — если `cond != 0`, возвращает `a`, иначе `b`
- `>`, `<`, `>=`, `<=`, `==`, `!=` — сравнения, возвращают `0.0` или `1.0`
- `&&` — логическое И
- `!` — логическое НЕ

Сенсоры:

- `temp()`
- `humidity()`

Цвет:

- `rgb(r, g, b)`
- `hsv(h, s, v)`

## Минимальный шаблон эффекта

```txt
effect "my_effect"

sprite dot {
	bitmap """
	#
	"""
}

layer paint {
	use dot
	color rgb(255, 120, 80)
	x = 10
	y = 6
	scale = 2
	rotation = 0
	blend = normal
	visible = 1
}
```

## Живой пример

```txt
effect "heart"

sprite heart {
	bitmap """
	.##.##.
	#######
	#######
	.#####.
	..###..
	...#...
	"""
}

layer love {
	use heart
	color rgb(255, 30 + 20 * sin(t * 5), 80)
	x = 10 + sin(t) * 4
	y = 4 + cos(t * 1.2) * 2
	scale = 1 + abs(sin(t * 2))
	rotation = t * 0.8
	blend = add
	visible = 1
}
```

## Что в Lux пока не поддерживается

- `compute`-блоки (per-pixel вычисления с `let`/`while`);
- рекурсия;
- пользовательские функции;
- произвольные скрипты;
- collision logic;
- логическое ИЛИ `||` (можно через `a + b - a*b`).

## Как писать Lux-эффекты без боли

Практический путь:

1. Начинай с одного `sprite` или `text`.
2. Добавляй один `layer`.
3. Сначала ставь фиксированные `x`, `y`, `scale`.
4. Только потом добавляй `sin`, `cos` и цветовую анимацию.
5. После этого проверяй и сохраняй preset.

Если сразу пытаться написать «огонь, молнию, радугу, шум и часы в одном файле», дебажить такой код потом бессмысленно.

## Рекомендации для LLM

LLM лучше работает, если задача поставлена жёстко. В запросе полезно явно фиксировать:

- размер полотна: `32x16`;
- желаемый образ эффекта;
- список разрешённых сущностей и функций;
- запрет на новые ключевые слова;
- требование вернуть только DSL-код без пояснений.

## `palette`

Палитра задаёт именованные цвета для использования в `sprite`. Вместо `rgb(...)` внутри bitmap можно писать короткие имена.

```txt
palette rainbow_pal {
  K = rgb(255, 0, 0)
  O = rgb(255, 127, 0)
  G = rgb(255, 255, 0)
}
```

Чтобы sprite использовал палитру, добавь `palette <имя>`:

```txt
sprite fwd_strip palette rainbow_pal {
  bitmap """
  K
  O
  G
  """
}
```

Каждый символ в bitmap — ссылка на цвет из палитры. Символ `.` — прозрачный пиксель (как обычно).

## `for`

Цикл `for` генерирует несколько layer-ов на этапе компиляции. На этапе исполнения цикла уже нет — есть готовые развёрнутые слои.

```txt
for k = 0; k < 32; k = k + 1 {
  layer wave {
    use dot
    color rgb(255, 255, 255)
    x = k
    y = 8 + 4 * sin(k * 0.2 + t * 3)
    scale = 1
    visible = 1
  }
}
```

**Правила `for`:**
- Переменная цикла — одно и то же имя во всех частях заголовка (инициализация, условие, шаг)
- Оператор сравнения: только `<` или `>` (оператор `<=` **не работает** — известный баг парсера)
- Границы цикла: **целые числа**, без выражений и переменных
- Шаг: `k = k + N` (составная форма) или `k = N` (простая). N — целое число
- Зарезервированные слова (`t`, `x`, `y`, `sin`, `rgb`, `hsv` и др.) нельзя использовать как переменную цикла
- Свойства внутри layer тела цикла должны быть **каждое на отдельной строке** (inline-формат `layer l { use dot  x = 5 }` не работает внутри for)

## `clock`

Блок `clock` управляет наложением часов поверх эффекта.

```txt
clock {
  enabled = 0
}
```

- `enabled = 0` — часы скрыты
- `enabled = 1` — часы показываются (по умолчанию)

## Аппаратные ограничения

ESP32-C3 имеет ограниченные ресурсы. При написании эффектов учитывай:

| Параметр | Лимит | Примечание |
|---|---|---|
| Размер исходного кода (JSON) | ~8 КБ | Больше — таймаут компиляции или обрыв HTTP |
| Слоёв после компиляции | ~200 | Больше — риск переполнения хипа |
| Глубина дерева выражений | 64 | Ограничение рекурсии evaluator-а |
| Глубина while в `compute` | 100 итераций | Аппаратный лимит |
| Размер спрайта (bitmap) | без жёсткого лимита | 32×16 = 512 символов — работает |
| JSON-буфер | 16 КБ | DynamicJsonDocument в куче |

**Что грузит лампу:**
- Много layer-ов с `for` — каждый слой = отдельный объект в памяти
- Глубокие цепочки `if(..., if(..., ...))` — растёт дерево выражений
- Сложные выражения с `sin`, `cos`, `%` в каждом пикселе

**Что работает быстро:**
- Один большой спрайт с готовой картинкой (bitmap 32×16) — 1 слой, мгновенная компиляция
- Несколько layer-ов с простыми выражениями
- `hsv(t * speed, 1, 1)` для цветовых переливов
- `x = expr + t * speed` для горизонтальной прокрутки (цилиндр зациклен по X)

## Известные баги парсера

- **`<=` в for:** оператор `<=` вызывает ошибку «Имя переменной цикла должно совпадать». Используй `<` с увеличением границы на 1.
- **Inline layer в for:** свойства layer-а должны быть на отдельных строках. Однострочный формат `layer l { use d  x = 5 }` не парсится внутри `for`.
- **Отрицательные границы в for:** отрицательные числа в границах цикла (`k = -6`) могут работать нестабильно. Лучше использовать `k = 0; k < N;` и прибавлять смещение в выражениях.
