# README and LLM Guidance Design

**Date:** 2026-03-23

## Goal

Собрать практичный README для mylamp, который покрывает:
- аппаратное подключение;
- базовую настройку и запуск;
- локальную frontend-разработку;
- текущее описание DSL для live coding;
- рекомендации по работе с LLM;
- готовый copy-paste prompt для генерации эффектов.

## Scope

README должен быть одновременно:
- входной точкой для человека, который впервые открывает репозиторий;
- честной документацией по текущему состоянию проекта;
- справочником по DSL v1;
- источником готового prompt-блока для LLM.

## Content Structure

1. Краткое описание проекта и текущий статус.
2. Поддерживаемое железо и схема подключения.
3. Быстрый старт для firmware.
4. Web UI и локальная разработка frontend.
5. Описание DSL:
   - effect
   - sprite + bitmap
   - layer + use/x/y/scale/color/visible
   - встроенные переменные
   - встроенные функции
   - ограничения языка
6. Практический раздел «Как создать новый эффект».
7. Раздел «Рекомендации по работе с LLM».
8. Большой fenced code block с copy-paste prompt.
9. Пример пользовательского запроса к LLM.
10. Пример ожидаемого ответа.

## Accuracy Rules

README должен явно разделять:
- то, что уже реально работает;
- то, что пока доступно только в dev/mock режиме;
- то, что ещё в процессе реализации.

Особенно важно не заявлять, что live validate/run уже полноценно реализованы на устройстве, если в firmware endpoints пока возвращают not implemented.

## Editing Strategy

- Переписать README целиком, сохранив ключевой контекст проекта.
- Оставить язык README русским.
- Синтаксис DSL описывать только по тем конструкциям, которые уже зафиксированы в frontend/editor docs.
- LLM prompt сделать строгим: перечислить допустимые конструкции и запретить выдуманные ключевые слова.
